# Slayer Task Planner

OSRS calculator: pick your slayer master (Duradel, Nieve, Konar, Krystilia) and see the expected **Slayer XP, loot and points** for every task you can be assigned.

- **Task planner** — every task with its assignment chance (rebalanced live as you block/lock tasks), average amount (extend flags), chosen kill, XP, GP and the % chance of at least one rare. Left-click a row to plan to skip it; flags per row: **E**xtended, **B**locked, **L**ocked (not unlocked / can't receive).
- **Kill choice** — global preference (base monster vs most profitable) with a per-task override dropdown: a spiders task can be Sarachnis, a gargoyle task can be Grotesque Guardians, a hellhound task can be Cerberus.
- **Points break-even** — long-run average points per completed task (10th/50th/100th/250th/1000th milestones included) minus the 30-point cost of your planned skips, with the break-even skip share.
- **Task breakdown** — enter your current task + amount for a full per-item loot table in category order (rares → alchables → herbs → runes → seeds → resources → supplies → other → tertiary). Click drops you leave on the ground to exclude them; "toggle rares" shows the task's GP as if you never hit a rare. Konar brimstone keys and Krystilia Larran's keys included as tertiary lines.
- Live GE prices from the wiki's real-time API with a baked snapshot fallback; in-game coin-stack colours; the actual RuneScape font.

## Data

`data.js` is generated by `node build-data.mjs` from the OSRS Wiki: master assignment tables (weights, amounts, extensions), every candidate monster's drop table (DropsLine parsing plus expansion of the wiki's shared herb/seed sub-table templates), infobox Slayer XP, and GE price mapping. Wiki responses are cached in `cache/` (gitignored); re-run the script any time to refresh rates. `build-warnings.txt` lists anything the parser couldn't handle.

Known simplifications: boss kills count 1:1 toward the task amount, Konar's per-location restrictions aren't modelled for loot, superiors aren't included, and clue scrolls/keys (tertiary) are shown but never added to GP totals.

No build step for the page itself — open `index.html` locally (it loads `data.js` from the same folder) or serve the folder statically.
